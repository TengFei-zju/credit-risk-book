# 第一章 信贷风控行业全景

## 1.1 信贷业务的本质

信贷业务的核心是**时间价值的交换**：金融机构以资金换取未来更多现金流，而风控的本质是识别和定价这笔交换的不确定性。

信贷风控要回答三个基本问题：

| 问题 | 模型方向 | 典型产品 |
|------|---------|---------|
| 借款人**愿不愿**还 | 意愿模型（反欺诈） | 欺诈评分、团伙识别 |
| 借款人**能不能**还 | 能力模型（信用风险） | 申请评分、行为评分 |
| 借款人**还多少** | 损失模型（LGD/EAD） | 催收评分、回收率预测 |

---

### 1.1.1 风险定价的基本原理

信贷风险定价的核心公式：

1184 Expected Loss = PD 	imes LGD 	imes EAD 1184

其中：
- **PD（Probability of Default）**：违约概率，通常定义为逾期 90 天以上的概率
- **LGD（Loss Given Default）**：违约损失率，违约后无法回收的比例
- **EAD（Exposure at Default）**：违约风险敞口，违约时的贷款余额

**风险定价的实践逻辑**：
1. 基于历史数据估计客户的 PD
2. 根据产品类型和担保方式估计 LGD
3. 根据授信额度和使用率估计 EAD
4. 计算期望损失，加上资金成本、运营成本、目标利润，得到最终利率

**监管资本要求**（巴塞尔协议）：
- 初级内部评级法（FIRB）：银行自行估计 PD，其他参数使用监管默认值
- 高级内部评级法（AIRB）：银行自行估计 PD、LGD、EAD

## 1.2 信贷产品分类

```
信贷产品
├── 消费信贷
│   ├── 现金贷（信用贷）：无抵押，依赖个人信用
│   ├── 分期贷：消费场景绑定
│   └── 信用卡：循环额度
├── 抵押贷款
│   ├── 房贷（按揭）
│   └── 车贷
├── 小微企业贷
│   ├── 经营贷
│   └── 供应链金融
└── 其他
    ├── 助贷/联合贷
    └── ABS/资产证券化
```

不同产品的风险特征差异显著，直接影响建模策略：

- **现金贷**：样本量大，特征维度高，欺诈风险突出，模型迭代快
- **房贷**：违约率低，样本稀少，抵押物价值是核心变量，监管要求高
- **小微贷**：财务数据造假风险高，替代数据（水电煤、流水）至关重要

## 1.3 风控的生命周期

一笔贷款从申请到结清，风控贯穿全程：

```
贷前              贷中              贷后
┌──────────┐   ┌──────────┐   ┌──────────────┐
│ 准入筛选  │→  │ 额度管理  │→  │ 早期预警/催收 │
│ 反欺诈    │   │ 行为监控  │   │ 核销/资产处置 │
│ 信用评估  │   │ 调额/降额 │   │ 回收率预测   │
└──────────┘   └──────────┘   └──────────────┘
申请评分        行为评分         催收评分
```

## 1.4 中国信贷风控的监管环境

### 1.4.1 核心监管机构

- **人民银行**：征信管理、利率上限（年化36%/24%红线历史演变）
- **银保监会（现国家金融监督管理总局）**：银行、消金公司监管
- **证监会**：ABS相关
- **地方金融监管局**：小贷公司

### 1.4.2 关键监管文件（风控算法工程师必知）

| 文件 | 核心要点 |
|------|---------|
| 《征信业管理条例》 | 数据采集边界、授权要求 |
| 《个人信息保护法》（PIPL） | 数据最小化、用户权利 |
| 《网络安全法》 | 数据本地化要求 |
| 《算法推荐管理规定》 | 算法可解释性要求 |
| 《商业银行互联网贷款管理暂行办法》 | 核心风控不得外包 |

### 1.4.3 对建模的影响

- **可解释性要求**：拒绝申请需提供可理解的原因，倒逼评分卡与可解释ML并存
- **数据合规**：三方数据接入需授权链完整，"共债"数据使用需谨慎
- **模型备案**：部分机构要求内部模型向监管报备

## 1.5 信贷风控的核心指标体系

### 1.5.1 资产质量指标

| 指标 | 定义 | 说明 |
|------|------|------|
| NPL（不良贷款率） | 不良余额/贷款总余额 | 监管关注核心指标 |
| M1+逾期率 | 逾期1期以上余额/期末余额 | 资产质量快照 |
| FPD（首逾率） | 首期逾期笔数/当期放款笔数 | 反映准入质量 |
| SPD（二逾率） | 第二期逾期 | 配合FPD看趋势 |
| Vintage（账龄分析） | 各放款月份的逾期率随账龄演变 | 贷后分析核心 |

### 1.5.2 模型性能指标

| 指标 | 适用场景 |
|------|---------|
| KS（K-S统计量） | 评分卡标配，衡量区分度 |
| AUC/GINI | 通用排序能力指标 |
| PSI（群体稳定性指标） | 模型监控，衡量分布漂移 |
| Lift/精确率 | 欺诈模型，衡量捕获效率 |

## 1.6 风控团队的组织结构

典型的风控团队分工：

```
风控部门
├── 政策与策略团队（风控经理）
│   └── 制定准入政策、额度政策、催收策略
├── 模型团队（算法工程师）
│   ├── 申请评分/行为评分
│   ├── 反欺诈模型
│   └── 催收模型
├── 数据团队
│   └── 数据治理、特征平台、标签体系
└── 系统团队
    └── 决策引擎、规则系统
```

**算法工程师的核心价值**：在数据与策略之间架桥，用量化模型替代经验拍板。

## 1.7 行业数据生态

### 1.7.1 官方征信

- **人行征信**：最权威，覆盖银行信贷记录，接入门槛高
- **百行征信**：互联网金融数据，覆盖网贷记录

### 1.7.2 商业数据源（需合规使用）

| 类型 | 代表机构/数据 | 核心价值 |
|------|-------------|---------|
| 运营商数据 | 三大运营商 | 通话行为、实名认证 |
| 电商数据 | 芝麻信用、京东 | 消费行为、还款记录 |
| 设备数据 | 同盾、瑞数 | 设备指纹、黑名单 |
| 社保公积金 | 各地社保局 | 收入稳定性证明 |
| 司法数据 | 最高法 | 失信被执行人 |

### 1.7.3 数据"沙漠化"趋势

2020年后强监管整顿导致大量三方数据源收紧，行业面临**数据沙漠化**挑战：
- 强化内部行为数据挖掘
- 重视申请页面埋点数据（鼠标轨迹、填表时长）
- 联邦学习等隐私计算方案探索

---

> **本章小结**：信贷风控不是单纯的技术问题，监管环境、数据生态、业务逻辑共同决定了建模空间的边界。理解这些约束，是做出真正落地方案的前提。
